سطح مبتدی (مفاهیم پایه)
🔹 DevOps چیه و چرا به وجود اومد؟ مبتدی
فرهنگ و روش‌کاری برای حذف دیوار بین Dev و Ops، افزایش سرعت تحویل و کاهش اصطکاک. هدف: رساندن ارزش به کاربر در کوتاه‌ترین زمان.
🔹 CI/CD مخفف چیه؟ مبتدی
CI = Continuous Integration (یکپارچه‌سازی مداوم)
CD = Continuous Delivery (تحویل مداوم) یا Continuous Deployment (استقرار مداوم).
🔹 تفاوت Docker و Kubernetes؟ مبتدی
Docker = ساخت و اجرای کانتینر (کانتینرایز کردن).
Kubernetes = مدیریت و هماهنگی کانتینرها در مقیاس بزرگ (ارکستراسیون). مکمل هم هستن.
🔹 تفاوت Docker Compose و Kubernetes؟ مبتدی
Compose = اجرای چند کانتینر روی یک ماشین (مناسب توسعه محلی).
Kubernetes = مدیریت کانتینرها روی کلاستر چند ماشین (مناسب Production).
🔹 تفاوت DevOps و DevSecOps؟ مبتدی
DevOps = Dev + Ops برای سرعت تحویل.
DevSecOps = DevOps + Security از ابتدای چرخه (Shift-Left).
🔹 استراتژی‌های دیپلوی رو نام ببر. مبتدی
  • Recreate (با DownTime)
  • Rolling (بدون DownTime)
  • Blue/Green (ریسک کم)
  • Canary (درصدی)
🔹 تفاوت تست Functional و Non-Functional؟ مبتدی
Functional = «چه کاری» انجام میده (مثل Unit, Integration).
Non-Functional = «چقدر خوب» انجام میده (مثل Performance, Security).
سطح متوسط (ابزارها و سناریوها)
⚡ تفاوت Deployment، StatefulSet و DaemonSet؟ متوسط
  • Deployment: برای اپ‌های بی‌حالت (Stateless).
  • StatefulSet: برای اپ‌های حالت‌دار (دیتابیس) با هویت ثابت.
  • DaemonSet: روی هر Node دقیقاً یک Pod (مثل مانیتورینگ).
⚡ Pod با وضعیت Pending، مراحل عیب‌یابی؟ متوسط
۱. kubectl describe pod
۲. چک: منابع ناکافی، PVC بسته نشده، ImagePullBackOff.
⚡ تفاوت Terraform و Ansible؟ متوسط
Terraform = Provisioning (ساخت زیرساخت).
Ansible = Configuration Management (تنظیمات داخل ماشین). مکمل هم.
⚡ زیرساخت تغییرناپذیر (Immutable) یعنی چی؟ متوسط
هیچ‌وقت سرور زنده رو Patch نمی‌کنیم؛ برای هر تغییر، سرور جدید می‌سازیم و قبلی رو می‌کشیم. Drift به صفر می‌رسد.
⚡ سه پایه‌ی Observability؟ متوسط
Metrics (اعداد) · Logs (رویدادهای متنی) · Traces (مسیر درخواست بین سرویس‌ها).
⚡ تفاوت Monitoring با Observability؟ متوسط
Monitoring: از پیش می‌دونیم چه متریک‌هایی رو ببینیم.
Observability: سیستم به‌قدری داده داره که هر سوال پیش‌بینی‌نشده‌ای رو جواب بده.
⚡ مدیریت Secrets در Kubernetes؟ متوسط
از آبجکت Secret، یا External Secrets Operator و HashiCorp Vault استفاده می‌کنیم. هرگز در Dockerfile یا کد نمی‌گذاریم.
⚡ اسکن امنیتی ایمیج‌ها با چی؟ متوسط
در CI از Trivy، Snyk یا Clair استفاده می‌کنیم. اگه آسیب‌پذیری Critical پیدا شد، Build رو Fail می‌کنیم.
⚡ تفاوت `git merge` و `git rebase`؟ متوسط
Merge: تاریخچه رو با کامیت جدید ترکیب میکنه (دقیق).
Rebase: تاریخچه رو خطی میکنه (تمیزتر). برای شاخه‌های شخصی از Rebase استفاده می‌کنم.
⚡ تفاوت REST و gRPC؟ متوسط
  • REST: JSON/XML، HTTP/1.1، مناسب APIهای عمومی.
  • gRPC: Protocol Buffer (باینری)، HTTP/2، پرسرعت، مناسب ارتباط داخلی میکروسرویس‌ها.
سطح حرفه‌ای (معماری، عیب‌یابی پیشرفته)
🧠 کل کلاستر Kubernetes افتاد، اول چه کار؟ حرفه‌ای
۱. چک etcd و Backup.
۲. ریستارت Control Plane.
۳. Join کردن Worker Nodeها. اگه etcd از بین رفت، از Backup Restore می‌کنیم.
🧠 پایپلاین باعث خرابی Production شد، چکار؟ حرفه‌ای
۱. فوری: Rollback با kubectl rollout undo.
۲. ذخیره‌ی لاگ و متریک.
۳. Post-Mortem بدون سرزنش + تست جدید به پایپلاین اضافه می‌کنیم.
🧠 الگوی Saga در میکروسرویس‌ها؟ حرفه‌ای
مدیریت تراکنش‌های توزیع‌شده با تراکنش‌های محلی و عملیات جبرانی. دو روش: Choreography (مبتنی بر رویداد) و Orchestration (مرکزی).
🧠 Service Mesh (مثل Istio) چیه؟ حرفه‌ای
لایه‌ی زیرساختی برای مدیریت ارتباطات بین سرویس‌ها. قابلیت‌ها: مدیریت ترافیک، امنیت (mTLS)، مشاهده‌پذیری و سیاست‌گذاری. با تزریق Proxy (Sidecar) کار می‌کنه.
🧠 API Gateway چیه؟ حرفه‌ای
دروازه‌ی ورودی واحد برای کلاینت‌ها. کاربردها: Routing, Authentication, Rate Limiting, Caching, Logging.
🧠 تفاوت IaaS, PaaS, SaaS؟ حرفه‌ای
  • IaaS: زیرساخت (EC2)
  • PaaS: پلتفرم (Heroku)
  • SaaS: نرم‌افزار کامل (Gmail)
🧠 شیوه‌ی عیب‌یابی مصرف بالای CPU در لینوکس؟ حرفه‌ای
۱. top یا htop ← پیدا کردن PID.
۲. strace -p PID ← بررسی سیستم‌کال‌ها.
۳. چک کردن لاگ‌های اپلیکیشن.
🧠 تفاوت SIGTERM و SIGKILL؟ حرفه‌ای
SIGTERM: مودبانه (اجازه‌ی پاک‌سازی میده).
SIGKILL: اجباری و فوری (بلافاصله می‌کشد). در K8s اول SIGTERM و بعد از Grace Period، SIGKILL.
🧠 معماری Monolithic vs Microservices؟ حرفه‌ای
  • Monolithic: ساده، دیپلوی آسان، مقیاس‌پذیری سخت.
  • Microservices: مقیاس‌پذیری مستقل، پیچیدگی شبکه و مدیریت دیتابیس توزیع‌شده.
🧠 Chaos Engineering چیه؟ حرفه‌ای
خراب‌کردن عمدی اجزای سیستم (مثل کشتن Pod) برای سنجش مقاومت و پیدا کردن نقاط ضعف. ابزار: Chaos Mesh, Gremlin.
➕ سوالات تکمیلی و ترکیبی
➕ Git Hook در DevOps؟ متوسط
اسکریپت‌هایی که در رویدادهای Git (مثل commit, push) اجرا میشن. کاربرد: Pre-commit برای Linter، Pre-push برای تست، Post-receive برای تریگر CI/CD.
➕ GitFlow vs Trunk-Based؟ متوسط
  • GitFlow: شاخه‌های متعدد (develop, feature, release) مناسب پروژه‌های بزرگ.
  • Trunk-Based: همه روی main کار می‌کنن با Branchهای کوتاه‌عمر، مناسب انتشار مداوم.
➕ Auto Scaling و Load Balancer در ابر؟ حرفه‌ای
Auto Scaling: افزایش/کاهش خودکار تعداد سرورها بر اساس بار.
Load Balancer: توزیع ترافیک بین سرورها. با هم مقیاس‌پذیری خودکار و بدون DownTime رو ممکن می‌کنن.
➕ تفاوت Object Storage و Block Storage؟ حرفه‌ای
  • Object (S3): فایل به‌صورت شیء با Metadata، دسترسی API، مناسب فایل‌های استاتیک.
  • Block (EBS): بلوک‌های خام، متصل به سیستمعامل، مناسب دیتابیس.
➕ سناریوی بحرانی: خطا در Production حرفه‌ای
۱. کشف: آلرت Prometheus.
۲. بررسی: چک Grafana, ELK, Tracing.
۳. اقدام: Rollback فوری.
۴. ریشه‌یابی: پیدا کردن علت (مثلاً کوئری بی‌ایندکس).
۵. پیشگیری: اضافه کردن تست Performance به CI و هشدار جدید.
➕ Reverse Proxy با Nginx؟ متوسط
سرور واسطه که درخواست‌ها رو به Backend هدایت میکنه. کاربردها: Load Balancing, SSL Termination, Caching, فیلتر کردن.

💡 نکات طلایی برای مصاحبه

  • هیچ جوابی را حفظ نکن؛ با یک مثال واقعی از کار خودت پررنگش کن.
  • اگه جواب سوالی رو نمی‌دونی، بگو: «روش حل کردنش اینه که فلان مستندات رو چک کنم و فلان لاگ رو ببینم.»
  • در انتها حتماً از مصاحبه‌کننده بپرس: «چطور ریسک انتشار رو اندازه‌گیری می‌کنید؟» و «بزرگ‌ترین چالش زیرساختی شما چیه؟»
  • این صفحه رو در Git خودت ذخیره کن و برای همه‌ی مصاحبه‌های آینده ازش استفاده کن.
