🔹 مبانی و فلسفه DevOps فرهنگ

DevOps چیه و چرا به وجود اومد؟
فرهنگ و روش‌کاری، نه نقش یا ابزار. برای از بین بردن دیوار بین Dev و Ops، افزایش سرعت تحویل و کاهش اصطکاک. هدف: رساندن ارزش به کاربر نهایی در کوتاه‌ترین زمان.
تفاوت SRE با DevOps چیه؟
DevOps = فلسفه · SRE = پیاده‌سازی عملی (گوگل). در SRE قابلیت اطمینان به SLA و Error Budget تبدیل می‌شود تا تصمیم بگیریم فیچر جدید بزنیم یا پایداری را بالا ببریم.

🔸 CI/CD قلب تپنده

تفاوت CI، CD (Delivery) و CD (Deployment)؟
  • CI: به‌ازای هر کامیت، Build + Test خودکار.
  • CD (Delivery): آماده‌ی انتشار روی Production، ولی نیاز به تأیید دستی دارد.
  • CD (Deployment): همه‌ی مراحل خودکار و انتشار بدون دخالت انسان روی Production.
استراتژی‌های بروزرسانی (Deployment Strategies)؟
  • Recreate: قدیمی را می‌کشیم، جدید را راه می‌اندازیم (بیشترین DownTime).
  • Rolling: یکی‌یکی Podها عوض می‌شوند (Downtime صفر، ولی نسخه‌ها مخلوط).
  • Blue/Green: دو محیط مجزا، ترافیک یک‌جا از Blue به Green هدایت می‌شود (ریسک کم).
  • Canary: درصد کمی ترافیک به نسخه‌ی جدید، سپس افزایش تدریجی.

🐳 کانتینر و Kubernetes ارکستراسیون

یک Pod چند Container می‌تونه داشته باشه؟
چندین Container (الگوی Sidecar). مثلاً Container اصلی + Container کناری برای لاگ، پراکسی یا Secrets که حافظه و شبکه مشترک دارند.
فرق Deployment، StatefulSet و DaemonSet؟
  • Deployment: برای اپلیکیشن‌های بی‌حالت (Stateless).
  • StatefulSet: برای اپلیکیشن‌های حالت‌دار (دیتابیس) با هویت ثابت.
  • DaemonSet: روی همه‌ی Nodeها دقیقاً یک Pod اجرا می‌کند (مثل مانیتورینگ).
Pod با وضعیت Pending، مراحل عیب‌یابی؟
۱. kubectl describe pod برای دیدن Events
۲. چک سه مورد اصلی:
  • منابع ناکافی (CPU/Memory)
  • PVC بسته نشده
  • ImagePullBackOff (مشکل در دریافت ایمیج)

🏗️ زیرساخت به‌عنوان کد (IaC) اتوماسیون

تفاوت Terraform و Ansible؟
Terraform = Provisioning (ساخت ماشین و شبکه) · Ansible = Configuration Management (نصب نرم‌افزار و تنظیمات داخل ماشین). هر دو مکمل هم هستند.
زیرساخت تغییرناپذیر (Immutable) یعنی چی؟
هیچ‌وقت سرور زنده را Patch نمی‌کنیم. برای هر تغییر، سرور جدید می‌سازیم، تست می‌کنیم و قبلی را می‌کشیم. Drift (انحراف) به صفر می‌رسد و دیباگ راحت‌تر می‌شود.

🌐 شبکه، لینوکس و عیب‌یابی عملیات

فرمان پیدا کردن فرآیند روی پورت ۸۰۸۰؟
sudo lsof -i :8080   یا   sudo netstat -tulpn | grep 8080
یک اپلیکیشن کند جواب میده، ترتیب عیب‌یابی؟
۱. CPU/Memory (top)  → ۲. Disk I/O (iostat)  → ۳. شبکه (Latency)  → ۴. لاگ‌های اپلیکیشن  → ۵. Tracing (Jaeger)

📊 مانیتورینگ و اوبزروبیلیتی دیدپذیری

تفاوت Monitoring با Observability؟
Monitoring: از پیش می‌دانیم چه متریک‌هایی را ببینیم (مثلاً CPU>۸۰٪).
Observability: سیستم به اندازه‌ای داده (لاگ، متریک، تریس) دارد که بتوانیم هر سوال پیش‌بینی‌نشده را جواب دهیم و بفهمیم چرا خراب شده.
سه پایه‌ی Observability؟
Metrics (اعداد) · Logs (رویدادهای متنی) · Traces (مسیر درخواست بین سرویس‌ها)

🔐 امنیت و مدیریت اسرار Secrets

مدیریت Password و API Key در Kubernetes؟
هرگز در Dockerfile یا کد نمی‌گذاریم. از آبجکت Secret یا External Secrets Operator و Hashicorp Vault استفاده می‌کنیم و به‌صورت Env یا Volume در اختیار Pod قرار می‌دهیم.
چطور از ایمیج‌ها در برابر آسیب‌پذیری محافظت می‌کنیم؟
در CI یک مرحله اسکن امنیتی با Trivy، Snyk یا Clair اضافه می‌کنیم و اگر آسیب‌پذیری Critical پیدا شد، Build را Fail می‌کنیم.

🚨 سناریوهای بحرانی نجات

کل کلاستر Kubernetes افتاد، اول چه کار؟
۱. چک etcd و Backup آن
۲. ریستارت یا Provision مجدد Nodeهای Control Plane
۳. Join کردن Worker Nodeها
(اگر etcd کاملاً از بین رفت، از Backup دیشب Restore می‌کنیم)
تغییر در پایپلاین باعث خرابی Production شد، چکار؟
۱. فوری: kubectl rollout undo یا برگرداندن Tag (Rollback)
۲. ذخیره‌ی لاگ و متریک برای پیدا کردن علت
۳. Post-Mortem بدون سرزنش + اضافه کردن تست جدید به پایپلاین

🤝 مهارت‌های نرم کلید پذیرش

توسعه‌دهنده می‌گه «کد من تو محیط خودم کار می‌کنه، نه تست»؛ چطور برخورد؟
به جای سرزنش، می‌گویم: «بیا با هم تفاوت‌های محیط رو چک کنیم.» اول Dockerfile و متغیرهای محیطی را مقایسه می‌کنم. پیشنهاد می‌دهم که از همان ایمیج استفاده کنیم و تفاوت‌ها را به ConfigMap ببریم.
چطور بین پایداری و فیچر جدید اولویت‌بندی می‌کنی؟
با Error Budget. اگر خطاهای سیستم کمتر از SLA است، بودجه داریم و می‌توانیم ریسک فیچر جدید را بپذیریم. اگر خطاها بالا رفته، اولویت با کاهش خطا و بدهی فنی است.
💡 نوک طلایی برای امروز: هیچ‌کدام از جواب‌ها را حفظ نکن؛ هرکدام را با یک مثال واقعی از کاری که خودت انجام دادی، پررنگ کن. مثلاً «من خودم یه بار با Canary deployment جلوی یه فاجعه رو گرفتم...» اینطوری ضریب قبولی ۹۰٪ می‌شود.
